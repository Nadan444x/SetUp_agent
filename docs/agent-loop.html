How SetUp Agent works

One page, ground-up: how a local LLM safely sets up your whole Mac from the terminal.

1 · The big picture

A script is a fixed list of commands. An agent looks at what just happened and chooses the next step. The safety trick: the LLM can only ask — Python decides.

🧠 Local LLM

qwen2.5 via Ollama, running on your Mac. Reads the goal + Setup.md, proposes one tool call at a time. Never touches the system.

→

🛡 Safety layer

Blocklist (rm -rf /…), no sudo, dry-run mode, and a y/N prompt showing the exact command.

→

⚙️ Executor

Runs the real command (brew install…, defaults write…) with a timeout, captures the output.

→

📄 Setup.md

The living system file: every success is recorded here with a timestamp — the recipe for your next laptop.

⤴ the result goes back to the LLM, which picks the next step — until nothing is left to do

3 · Why it can't wreck your machine

Blocklist

rm -rf /, disk formatting, dd to devices, fork bombs, curl | sh → refused. The refusal is sent back to the model so it changes plan instead of crashing.

No sudo, ever

Homebrew, git, defaults, shell config — everything the agent does works without root. Any sudo request is refused.

Dry-run

--dry-run shows every command it would run and executes nothing. Preview a whole machine setup risk-free.

You approve each change

Every mutating command is printed and waits for y/N. --yes only auto-approves routine brew/git/defaults commands.

4 · The two layers on a fresh Mac

0
bootstrap.sh — dumb shell, one commandA bare Mac has no brew, no Python, no Ollama — the agent can't install its own brain. This script installs exactly those prerequisites (skipping what exists), then hands off.
1
setup-agent scanReads the machine (read-only) and generates Setup.md: apps, dev tools, runtimes + versions, git identity, macOS prefs.
2
setup-agent setupThe LLM walks Setup.md section by section: checks each item, installs the gaps, applies git/shell/macOS settings — updating the file as it goes.
3
Forever afterAnything you install through the agent (run / chat) is recorded automatically. When the next laptop comes, your Setup.md is already the recipe.
